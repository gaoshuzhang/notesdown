后记

这本书是在 RStudio 内用 R Markdown 写的， Git 控制版本， bookdown 组织章节， knitr 将 R Markdown 源文件转化为 Markdown 文件， Pandoc 再将 Markdown 文件转化为 HTML 文档，而要转化为 PDF 文档则另外需要 TinyTeX 发行版2 和来自 rticles (Allaire et al. 2017) 包的模板（一个基于 Pandoc 的 LaTeX 模板，两个美元符号包含的就是变量，变量基本上是 LaTeX 包的设置选项）。这个网站是通过 Travis-CI 把编译结果（即 _book 目录）推送到 Netlify，实现部署。在 Travis-CI 和 Netlify 都与 Github 绑定的情况下，源代码一发生改变就会触发编译和部署，即持续集成和连续部署，你正在阅读的是 2018-05-18 在 Travis 上构建的。

如果编译和部署成功，你会看到一个绿色的指示器，否则指示器显示红色。

sourceserifpro 设为默认英文字体，inconsolata 用于代码显示，R 包名称在文中以粗体显示，代码块输出用 #> 表示，以区分普通的代码注释 #

我用了两个 R 包编译这本书，分别是 knitr (Xie 2015) 和 bookdown (Xie 2016)。绘图使用的中文字体是思源宋体和思源黑体， showtext 包安装和调用，tikzDevice 和 fontcm 处理其中的数学公式，xkcd 设置漫画手写体风格，以下是我的 R 进程信息：
